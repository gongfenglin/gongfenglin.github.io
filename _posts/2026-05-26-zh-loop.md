### 从一次性输出到运行循环

我说的 **Loop**，不是把模型调用包一层循环那么简单。它更像 Agent 的工作节奏。

一个可靠的 Agent 不应该只完成一次回答，而应该在一个循环里运行：定义任务、收集上下文、制定计划、调用工具、验证结果、记录过程，再把结果反馈到下一轮。每一轮都应该让下一轮更少盲区。

### 验证是 Loop 的核心

很多 Agent 系统的问题，是 loop 只做到 act 就结束了。真正重要的是 verify。

对 coding agent 来说，验证可以是测试、构建、diff、lint、benchmark 或人工 review gate。对 LLM eval 来说，验证可以是 assertion suite、provider-level xfail、XPASS 报告和 trace 检查。对文档管线来说，验证可以是转换质量、延迟、重试和失败恢复。

### 为什么我一直关注它

Loop 把 Agent runtime 和 Eval 连接起来。它把记忆、工具、子 Agent、benchmark 和 release gate 放进同一个节奏里。

我现在做的很多事情，都是在把这个 loop 显式化，让 Agent 的改进不再依赖零散的感觉，而是可以系统迭代。
